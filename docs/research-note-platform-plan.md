# 연구노트 관리 플랫폼 구축 기획서

## 1. 프로젝트 개요

### 1.1 프로젝트명
- 연구노트 관리 플랫폼 구축

### 1.2 추진 배경
조직 단위 프로젝트에서 연구기록, PDF 문서, 이미지 자료를 체계적으로 관리하기 위한 플랫폼이 필요하다. 기존 방식은 자료가 파일 단위로 분산되고 프로젝트·연구노트·첨부자료 간 연결성이 낮아 기록 연속성, 조회 편의성, 운영 효율성이 떨어진다.

### 1.3 개발 목적
- 조직 단위 연구 프로젝트 관리 체계 구축
- 프로젝트 하위 연구노트 작성 및 관리
- PDF 업로드 시 페이지 단위 자동 분해 및 관리
- 이미지 자료의 직관적 등록 및 관리
- 조직 내 권한 기반 접근 제어
- 감사 로그 기반 기록 추적성 확보
- 향후 전자결재, OCR, 검색, AI 기능 확장 가능한 구조 확보

---

## 2. 시스템 목표

### 2.1 관리 목표
- 조직별 사용자/프로젝트 분리 관리
- 프로젝트별 연구노트 독립 CRUD
- 연구노트 첨부 PDF/이미지의 구조적 관리

### 2.2 기록 목표
- PDF 업로드 시 자동 페이지 분해 저장
- 이미지 업로드 시 개별 단위 관리
- 생성/수정/업로드 등 이력 기록

### 2.3 기술 목표
- 백엔드: Python + FastAPI
- 아키텍처: DDD (Domain-Driven Design)
- 프론트엔드: TypeScript + React
- 프론트엔드 기본 구조: `api`, `pages`, `components`, `layout`
- 확장성과 유지보수성을 고려한 계층형 구조 적용

## 3. 주요 사용자 및 권한 구조

### 3.1 전역 권한
- `system_admin`: 전체 시스템 관리
- `user`: 일반 사용자

### 3.2 조직 내 권한
- `owner`: 조직 최고 권한자
- `admin`: 조직 운영 관리자
- `member`: 일반 사용자

### 3.3 권한별 역할

#### owner
- 조직 생성 및 관리
- 조직 멤버 권한 관리
- 프로젝트 생성 및 관리
- 조직 전체 연구노트 접근

#### admin
- 프로젝트 관리
- 연구노트 관리
- 조직 운영 대행
- 멤버 관리 보조

#### member
- 프로젝트 참여
- 연구노트 작성/수정
- PDF/이미지 업로드

## 4. 핵심 기능 범위

### 4.1 사용자 및 조직 관리
- 사용자 계정 관리
- 조직 생성
- 조직 가입 코드 기반 참여
- 조직 멤버 권한 설정

### 4.2 프로젝트 관리
- 조직 하위 프로젝트 생성
- 프로젝트 기본정보 관리
- 프로젝트 참여자 관리

### 4.3 연구노트 관리
- 프로젝트 하위 연구노트 생성
- 연구노트 제목/본문 작성
- 연구노트 수정/삭제
- 프로젝트별 연구노트 목록 조회

### 4.4 파일 관리
- 연구노트별 원본 파일 관리
- 지원 파일 유형: PDF, 이미지
- 업로드 파일 메타정보 저장

### 4.5 PDF 페이지 분해
- PDF 업로드 시 자동 페이지 분해
- 각 페이지를 이미지 단위로 저장
- 페이지 순서 관리

### 4.6 이미지 업로드
- 이미지 1장 단위 업로드
- 연구노트 연결
- 화면 표시 단위 관리

### 4.7 감사 로그
- 사용자 주요 행위 기록
- 생성/수정/삭제/업로드 이벤트 추적

## 5. 시스템 구조

### 5.1 데이터 구조
- 조직(`company`)
- 조직 멤버(`company_member`)
- 프로젝트(`project`)
- 프로젝트 멤버(`project_member`)
- 연구노트(`research_note`)
- 연구노트 파일(`research_note_file`)
- 연구노트 페이지(`research_note_page`)
- 감사 로그(`audit_log`)

### 5.2 파일 처리 구조
- PDF 업로드: 원본 저장 → 페이지 분해 → 페이지 이미지 저장
- 이미지 업로드: 원본 저장 → 단일 페이지 등록

## 10. 비즈니스 규칙
- 하나의 프로젝트는 반드시 하나의 조직에 속한다.
- 하나의 연구노트는 반드시 하나의 프로젝트에 속한다.
- 하나의 연구노트 파일은 반드시 하나의 연구노트에 속한다.
- PDF 업로드 시 페이지 단위 데이터가 생성되어야 한다.
- 이미지 업로드 시 1개의 페이지 데이터가 생성되어야 한다.
- 조직에는 최소 1명의 owner가 존재해야 한다.
- 조직 멤버만 해당 조직 프로젝트에 참여할 수 있다.
